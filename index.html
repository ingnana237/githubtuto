Cheat Sheet Versionner son travail Naviguer dans Git Bash pwd mkdir “dossier” touch fichier.txt ls ls -la cd dossier cd .. savoir dans quel dossier je suis créer un dossier (Make Directory) créer fichier liste le dossier courant liste tout plus précisément que ls aller dans le dossier (Change Directory) Remonter d’un dossier Initialisation de Git git config --global user.name git config --global user.email git config --global --list “Mon Nom” mon@mail.com Affiche nom et mail Commit son projet sur Github git add . git commit -m “message” git push -u origin master Autres commandes git status git diff état du fichier affiche les modifs avant commit Versionner en local git init git add . git commit -m “explication” initialise le dépôt (se mettre sur le bon dossier), mieux à faire depuis Github.com ajoute toutes les modifications (le . symbolise tout) créer un nouveau commit. git add pousse les fichiers en zone d’index, git commit les sauvegarde réellement dans un nouveau commit Gérer les commits git log git log -n2 git show sha-1 git checkout sha-1 git checkout master liste des commits affiche les 2 derniers commits voir commit spécifique (cliquer molette souris pour coller) remettre la version du sha-1 remettre version la plus récente Versionner sur un dépôt distant git clone lien-github.com git push -u origin master git push -f origin master récupérer travail depuis dépôt distant pousse les modifications vers serveur pousse de force des modifications (à manipuler avec précaution)
